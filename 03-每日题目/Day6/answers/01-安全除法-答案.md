# 01 - 安全除法 - 答案

## 完整代码

```python
def safe_divide(a, b):
    """安全除法：用 try/except 处理除零和类型错误
    
    参数:
        a: 被除数
        b: 除数
    
    返回:
        成功时返回 float 结果，失败时返回错误提示字符串
    """
    try:
        return a / b
    except ZeroDivisionError:
        return "错误：除数不能为零"
    except TypeError:
        return "错误：请输入数字"


# ========== 测试 ==========
if __name__ == "__main__":
    # 正常除法
    print(safe_divide(10, 2))       # 5.0
    print(safe_divide(7, 3))        # 2.333...
    print(safe_divide(-10, 4))      # -2.5
    
    # 除零
    print(safe_divide(10, 0))       # 错误：除数不能为零
    print(safe_divide(0, 0))        # 错误：除数不能为零
    
    # 类型错误
    print(safe_divide(10, "a"))     # 错误：请输入数字
    print(safe_divide("hello", 2))  # 错误：请输入数字
    print(safe_divide([1, 2], 3))   # 错误：请输入数字


# ========== 思考：except 的顺序 ==========
"""
Python 按顺序检查 except 子句，第一个匹配的会被执行。

正确的顺序（具体 → 通用）：
    except ZeroDivisionError:   # 先捕获具体异常
    except TypeError:            # 再捕获其他具体异常
    except Exception:            # 最后捕获通用异常

错误的顺序：
    except Exception:            # 先写了通用异常
    except ZeroDivisionError:    # 永远不会被执行！因为 ZeroDivisionError 是 Exception 的子类
"""
```

## 关键要点

1. **`try/except` 结构**：`try` 块中放可能出错的代码，`except` 块中处理异常
2. **异常类型**：
   - `ZeroDivisionError`：除数为零
   - `TypeError`：操作数类型不支持（如数字除以字符串）
   - `ValueError`：值不合法（如 `int("abc")`）
3. **捕获顺序**：从最具体的异常到最通用的异常，否则具体异常可能永远不会被触发
4. **异常继承关系**：`ZeroDivisionError` 和 `TypeError` 都是 `Exception` 的子类
5. **vs 提前 if 判断**：`try/except` 和 Day4 中 `if b == 0` 的区别是——`if` 只能预判已知的错误条件，`try/except` 能捕获所有运行时异常
