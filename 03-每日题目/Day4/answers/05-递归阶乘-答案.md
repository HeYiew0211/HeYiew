# 05 - 递归阶乘 - 答案

## 完整代码

```python
# ========== 递归版本 ==========
def factorial(n):
    """递归计算 n 的阶乘
    
    递归思想：n! = n × (n-1)!
    基准条件：0! = 1
    """
    # 基准条件：递归的终止点
    if n == 0:
        return 1
    # 递归调用：n! = n × (n-1)!
    return n * factorial(n - 1)


# ========== 循环版本 ==========
def factorial_loop(n):
    """用循环计算 n 的阶乘"""
    result = 1
    for i in range(1, n + 1):
        result *= i  # 等价于 result = result * i
    return result


# ========== 测试 ==========
if __name__ == "__main__":
    test_values = [0, 1, 5, 10]
    
    print("n\t递归版\t\t循环版")
    print("-" * 35)
    for n in test_values:
        r1 = factorial(n)
        r2 = factorial_loop(n)
        print(f"{n}\t{r1}\t\t{r2}")
    
    # 验证两个版本结果一致
    print("\n=== 验证两个版本是否一致 ===")
    for n in range(21):  # 测试 0 到 20
        assert factorial(n) == factorial_loop(n), f"n={n} 时不一致！"
    print("所有测试通过！递归版和循环版结果完全一致。")
```

## 关键要点

1. **递归两个要素**：
   - **基准条件（Base Case）**：`n == 0` 时返回 1，递归在此终止
   - **递归调用**：`n * factorial(n-1)`，问题规模逐步缩小
2. **递归执行过程**（以 `factorial(3)` 为例）：
   ```
   factorial(3) → 3 * factorial(2)
                  → 3 * (2 * factorial(1))
                  → 3 * (2 * (1 * factorial(0)))
                  → 3 * (2 * (1 * 1))
                  → 6
   ```
3. **递归 vs 循环**：
   - 递归：代码更简洁优雅，但深层递归会消耗栈空间
   - 循环：更高效，不会栈溢出，适合大规模数据
4. **`assert` 断言**：用于测试，条件为 False 时抛出异常 — 是验证代码正确性的好工具
